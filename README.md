## 2) Краткое описание игры, реализации и точек входа

Проект представляет собой минималистичный бесконечный 3D-раннер.  
Игрок автоматически движется вперед по коридору, может смещаться влево/вправо и прыгать (включая двойной прыжок).

### Основная логика
- Игрок постоянно движется вперед с базовой скоростью.
- Смещение по оси X управляется через новую Input System.
- Прыжок реализован через Rigidbody импульсом вверх.
- Ограничено количество прыжков в воздухе (maxJumps), счетчик прыжков сбрасывается при касании земли.
- При столкновениях с препятствиями игрок получает урон.
- При достижении 0 HP выполняется перезапуск игры.
- Препятствия спавнятся периодически перед игроком, с случайным типом и позицией по линиям.
- Старые препятствия удаляются после прохождения игроком.
- Скорость игры со временем увеличивается (ускорение сложности).

### Точки входа
- Главная сцена: Assets/Scenes/MainScene.unity
- Ключевые скрипты:
  - Assets/Scripts/Player Scripts/PlayerMovement.cs - движение/прыжки/ввод игрока
  - Assets/Scripts/GameManagerScripts/GameSpeedController.cs - управление текущей скоростью игры (ускорение)
  - Assets/Scripts/Obstacle Scripts/ObstacleSpawner.cs - генерация препятствий
  - Assets/Scripts/Player Scripts/PlayerHealth.cs - здоровье, урон, смерть/рестарт

> Примечание: все основные параметры (скорости, урон, HP, частота спавна и т.д.) настраиваются через Inspector.

## 3) Реализованный функционал по требованиям ДЗ2

### 3.1 Новый функционал (бонусы + очки)

#### Бонусы
Реализовано 2 типа бонусов:
- `Heal` (восстановление здоровья)
- `Invulnerability` (временная неуязвимость)

Как это сделано:
- На сцене спавнятся бонус-объекты.
- При подборе бонуса у игрока вызывается соответствующий метод в системе здоровья/эффектов:
  - для `Heal` - увеличение текущего HP с ограничением `maxHealth`,
  - для `Invulnerability` - включение флага неуязвимости на заданное время.
- Для повторного подбора временного бонуса используется обновление таймера эффекта.

#### Счётчик очков
Реализован счёт за время выживания:
- Очки накапливаются в игровом менеджере каждый кадр по `Time.deltaTime`.
- Значение переводится в целое и отображается в HUD.

---

### 3.2 Главное меню и HUD

#### Главное меню
В меню отображаются:
- название раннера,
- имя разработчика,
- рекорд игрока,
- кнопки `Play` и `Quit`.

Как реализовано:
- Через Canvas и панели (`MainMenuPanel`, `HUDPanel`, `PausePanel`).
- Управление видимостью панелей и состоянием игры выполнено в `UIManager`.
- При старте сцены активна только панель главного меню.

#### HUD во время игры
Отображаются:
- текущее здоровье,
- текущий счёт,
- рекорд.

Как реализовано:
- `UIManager` обновляет TMP-тексты в `Update` (только когда игра в состоянии `Playing`).
- Значения берутся из игровых скриптов (`PlayerHealth`, менеджер).

#### Пауза
Реализовано меню паузы:
- открытие по `ESC`,
- кнопки `Resume` и `Back to Menu`,
- остановка времени при паузе.

Как реализовано:
- Через новую Input System (`Pause` action) и метод обработки паузы.
- При паузе выставляется `Time.timeScale = 0`, при продолжении - `1`.
- Дополнительно блокируется управление игроком и спавнер на время паузы.

---

### 3.3 Система сохранения

Реализовано сохранение рекорда между запусками.

Как реализовано:
- Используется `PlayerPrefs`:
  - чтение рекорда при старте,
  - обновление при превышении текущим результатом,
  - сохранение через `PlayerPrefs.Save()`.
- Рекорд выводится в главном меню и в HUD.

---

### 3.4 Анимации (дополнительное требование)

Добавлены анимационные отклики через Unity Animator:
- на урон (`TakeDamage` trigger),
- на подбор бонуса (`Heal`/`TakeHeal` trigger),
- на длительный эффект (`IsInvulnerable`/`IsBuffActive` bool, состояние loop).

Как реализовано:
- На игроке создан `Animator Controller` со state-машиной (`Idle`, `TakeDamage`, `Heal`, `BuffLoop`).
- Переходы настроены через параметры и условия.
- В коде добавлен слой вызова анимаций, который выставляет `Trigger/Bool`.
